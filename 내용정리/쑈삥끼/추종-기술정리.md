# 쑈삥끼(ShopPinkki) — 추종(Tracking) 기술 정리

> Pinky Pro 로봇이 고객의 인형을 인식·추적해 따라다니는 스마트 쇼핑카트 데모의 **추종 파이프라인** 기술 정리.
> 코드(`shoppinkki_perception`, `ai_service/yolo`, `bt_tracking.py`, `config.py`)와 발표 슬라이드(33~39, 67~70) 양쪽에서 교차 확인함.

---

## 개요

추종은 **"인형을 찾고 → 계속 추적하고 → 주인을 식별하고 → 따라가는"** 4단계 파이프라인이다.
기성 인형 인식 모델이 없어 **데이터셋 구축부터 학습·엣지(Pi 5) 배포까지 직접 수행**했다.

| 단계 | 핵심 기술 |
|---|---|
| ① 탐지 | 커스텀 YOLOv8 (SAM3 라벨링 · 300장 학습) |
| ② 추적 | ByteTracker (Kalman + Hungarian + IoU) |
| ③ 재식별 | ReID(OSNet, 512D 코사인) + HSV 히스토그램 상관계수 |
| ④ 제어 | bbox 기반 PI(선속도) / P(각속도) 제어 |
| ⑤ 배포 | NCNN 경량 추론 (Raspberry Pi 5) |

---

## 1. 탐지 — 커스텀 YOLOv8 (직접 학습)

- 기성 인형 인식 모델이 없어 **직접 데이터셋을 구축해 훈련**.
- **Meta SAM3(Segment Anything)** 로 인형만 반자동 라벨링 → **300장 데이터셋** 생성 → 인형 전용 YOLOv8 학습.
- 데이터셋 구축(라벨링 도구 활용)부터 학습·배포까지 직접 수행.

## 2. 추적 — ByteTracker (다중 객체 추적, MOT)

- 프레임 간 동일 객체에 `track_id`를 부여·유지.
- 코드 적용: `results = model.track(img, persist=True)` → `track_id = int(det.boxes.id[0])`
- 내부 원리까지 학습해 정리:
  - **Kalman Filter** — 다음 프레임 위치 예측
  - **Hungarian Algorithm** — 트랙 ↔ 신규 bbox를 전체 비용 최소로 1:1 최적 매칭
  - **IoU 비용 행렬** — 겹침 정도 기반, `cost = 1 − IoU`

## 3. 재식별 — ReID + HSV 히스토그램 (2단계 검증)

**변화하는 `track_id` 속에서 "주인 인형"을 매 프레임 식별하기 위한 매칭 로직.**

- **ReID (Re-Identification)**
  - 라이브러리 `torchreid`, 모델 **OSNet x0.25** (Pi 5에서 실행 가능한 경량 모델)
  - 이미지를 **512차원 특징 벡터**로 변환 → **코사인 유사도**로 등록된 주인 인형과 비교
- **HSV 히스토그램 상관계수** (보조 검증)
  - `cv2.calcHist`로 H/S/V 각 16 bin → **48차원** 색 분포 벡터
  - 색 분포를 **상관계수**로 비교, ReID의 오인식(비슷하게 생긴 다른 인형)을 색상으로 보완
  - HSV는 밝기(V)가 분리돼 **조명 변화에 강인**
- **판정: "특징도 비슷(ReID) + 색도 비슷(HSV)" AND 조건**
  - 코드: 코사인·HSV 상관계수 임계값을 **동시 통과** + **5프레임 연속** 매칭 시 대상 락(`safe_id` 빠른 경로)
- 엣지(Pi 5) 대응: 경량 OSNet 선택 + CNN 실패 시 색상 통계로 폴백 설계

### 참고 원리
- **코사인 유사도** `cos(θ) = A·B / (|A||B|)` — 크기(밝기·거리) 무시, 방향(특징)만 비교. 512차원 공간에서도 동일 원리.
- **HSV 히스토그램** — 픽셀 색상을 bin에 나눠 분포화. 같은 인형이면 프레임이 바뀌어도 분포 모양 유지.

## 4. 제어 — bbox 기반 PI / P 제어 (직접 튜닝)

`config.py`로 검증 (`KP_DIST`, `KI_DIST` 존재, **`KD = 0`**).

- **선속도 (linear_x) — PI 제어**
  - `linear_x = KP_DIST × error + KI_DIST × ∫error·dt`
  - bbox 크기로 거리 추정: `TARGET_SIZE`보다 작으면(멀면) 전진, 크면(가까우면) 감속
  - **I항 사용 이유**: P만 쓰면 가까워질수록 오차·속도가 줄어 **바닥 마찰 때문에 목표 직전에 멈춤(정상상태 오차)**. I가 작은 오차를 누적해 마찰을 이길 힘을 만듦.
- **각속도 (angular_z) — P 제어**
  - `angular_z = KP_ANGLE × (화면중심X − bbox중심X)`
  - **±45px deadzone**: 중앙 근처 미세 진동에 반응 안 함
  - **I 제외 이유**: 각도 오차 누적 시 I항이 목표 초과 회전(오버슈팅) → 좌우 진동 유발
  - **low-pass 필터**: 신값 30% + 이전값 70%로 섞어 급격한 변화 억제
- **D를 뺀 공통 이유**: bbox 크기·중심 좌표는 프레임마다 노이즈가 커서, 변화율에 반응하는 D가 과반응하면 속도가 떨림 → 의도적으로 PI/P만 사용.

## 5. 배포 — 엣지 최적화

- 서버 YOLO(TCP:5005)와 **Pi 5 로컬 NCNN 경량 추론**을 병행해 실시간성 확보.

## 이력서/포트폴리오용 문장

**서술형 (핵심역량용)**

> **AI 기반 객체 추종 시스템 (ROS 2 / Python)** — 기성 모델이 없던 인형 인식을 위해 **Meta SAM3로 데이터셋을 구축·라벨링해 커스텀 YOLOv8을 직접 학습**했고, **ByteTracker(MOT) + OSNet ReID(512D 코사인 유사도) + HSV 히스토그램 상관계수**를 결합해 유사 객체 속에서 주인을 식별하는 파이프라인을 구현했다. 추종 제어는 **bbox 기반 PI/P 제어기를 직접 튜닝**하되, 영상 노이즈 특성을 근거로 D항을 배제하고 deadzone·low-pass 필터를 적용해 진동 없는 주행을 달성했다.

**한 줄 압축형**

> 처음 접한 **YOLOv8·SAM3·ByteTracker·ReID·PID 제어**를 단기간에 학습해, 커스텀 데이터셋 구축부터 엣지(Pi 5) 배포까지 실제 추종 시스템에 적용.

**정량 근거 관점 (성능 측정·비교 역량용)**

> D항 유무에 따른 속도 진동, ReID 단독 vs ReID+HSV 오인식률 등 **데이터에 근거해 제어·인식 방식을 취사선택**했다.
