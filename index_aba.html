Overview01 / 08
프로젝트 주제

사서를 대체하지 않고 사서의 손과 다리가 되는 협동로봇

주행로봇+로봇팔로 서가에서 책을 직접 집어 이송·분류·진열하고, 이용자를 추종해 짐을 옮기는 사서 협동로봇.

사용 기술
ROS2 JazzyNav2SLAMQt QML · C++FMS
프로젝트 배경
  • 사서의 반복 업무(배달·반납·정렬·재고 확인) 과중 — 핵심 업무 집중 필요
  • 야간 무인 시간대의 관리·보안 공백 — 침입 감지
  • 주행·매니퓰레이션·AI를 엮은 서비스형 로봇의 실사용 검증 필요
팀 구성 · 내가 맡은 부분
6인 팀 (Team ARTE) · 본인 팀장 · 주 1회 스프린트 운영

  • LiBi GUI — Qt QML + C++ 로봇 터치패널 구현

Software Architecture — 3계층 (Client · Server · Equipment) · HTTP · WS · ROS2(DDS) · TCP/UDP

  • 인식 (TCP·UDP) — 카메라 영상 → AI Server 도서 인식(book_detector→classifier)·추종 추론, Libi는 주행에 집중

State Diagram

상태 다이어그램

State03 / 08
FSM — Operation States + Task States, 전이 트리거 표기 (설계 · 본인)
Operation States (운영)
  • CHARGING — 충전, 배터리 ≥80% 시 IDLE
  • IDLE — 대기, 작업·순찰 명령 수신
  • PATROL / SECURITY_PATROL — 정기 · 야간 보안 순찰
  • RETURNING — 복귀, docked 시 충전
  • ERROR — fault · estop 즉시 정지, restart 복구
Task States (작업)
  • GUIDE 길잡이 · TRANSPORT 이송 · SORTING 분류 · FOLLOW 추종(짐꾼)
주요 전이 — battery_low(≤15%) · patrol_request(auto/manual) · task_request / task_done · fault·estop → ERROR
