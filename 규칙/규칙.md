# 취업준비

이력서 vs 자기소개서 vs 포트폴리오

이력서

- 학교, 자격증, 교육, 경력, 대외활동, 수상, 어학성적

자기소개서

- 지원동기, 성격의 장/단점, 본인의 강점/약점, 위기 해결 경험, 입사 후 포부

포트폴리오

- 얼마동안 수행했는가?
- 어떻게 해결했는가?
- 어떤 기여를 했는가?
- 어떤 과정을 거쳤는가?
- 어떤 결과를 냈는가?

**자기소개서**

좋은 자소서 특징

- 두괄식
- 숫자사용
- 문제 해결 중심
- 직무 연결

자소서는 스토리링텔링이 아니라 문제 해결 보고서이다.

자소서 프로젝트 관련 내용 구조 공식

- 결 : 내가 만든 결과
- 결론 → 경험 → 결과 순으로
- 결 → 기 → 승 → 전 → 결
    - 기 : 어떤 상황이었는가?
    - 승 : 어떤 문제가 있었는가?
    - 전 : 내가 무엇을 하였는가?
    - 결 : 그래서 무엇이 달라졌는가?, 무엇을 느꼈는가?, 그걸 어떻게 써먹을 것인가?

자기소개서에 포함하면 좋을 것들

- 자기소개
    - 내가 어떤 엔지니어인지 한 줄 정의
    - 핵심 강점 1개
    - 그 강점의 근거
    - 입사 후 방햔
- 지원동기
    - 기업 이해
    - 직무 이해
    - 내가 연결되는 지점

만약 실패 관련 내용을  말할 때

- 남 탓 금지
- 배운 점 강조
- 재발 방지 설명 (아이디어 제시)

포트폴리오

기업에서 중요 시 하는 것

- 기술 수준
- 문제 해결 능력
    - 문제 해결 구조 STAR
        - S : 어떤 상황이었는가?
        - T : 어떤 문제에 봉착했는가?
        - A : 어떤 행동을 했는가?
        - R : 어떤 결과가 있었는가?
- 사고 과정
    - 어떤 문제가 있었는가
    - 어떻게 접근했는가
    - 어떤 선택을 했는가
    - 결과는 무엇인가
    - 무엇을 고민했는가
- 커뮤니케이션 방식
- 성장 가능성
- 문제를 정의하는 능력
    - 이때의 접근 전략
- 의사결정 기준
- 트레이드 오프 이해
    - 선택지에는 항상 포기가 있다. 그걸 이해하고 있다는 걸 보여줘야한다.
- 회고 능력

사소하지만 치명적인 요소 ( 다하고 점검 필수)

- 오탈자
- 맞춤법
- 문장 완성도
- 링크 접근 오류
- 파일 깨짐(이미지 오류, 폰트 깨짐 등)

프로젝트 구조화 ( 프로젝트 담당 부분 상세 설명 / 문제해결부분)

- 상황 - 문제 - 행동 - 결과
    - 기승전결이 있는 글이 약 49% 상대 개선(읽기 쉬움)

포트폴리오의 프로젝트 작성 구조

- 기간
- 프로젝트 배경
- 프로젝트 목표
- 성과/기여한 점
- 사용 기술
- 결과/기대효과
- 팀 구성 / 맡은 책임

포트폴리오 전체 구조(ppt)

1. 인트로
2. 목차
    1. 흐름설계
    2. 페이지 넘버링 필수
    3. 대표작을 앞에 배치
3. 프로필
    1. 요약 소개
        1. 이력서의 내용을 요약 하여
4. 프로젝트
    1. 프로젝트 작성 구조로
    2. 각 프로젝트들 배경 / 사용기술 등 간단한 프로젝트 설명
    3. 프로젝트 상세설명
        1. 내가 한 것들 소개
        2. 문제 해결
5. 회고
    1. 회고가있는 사람은 성장하는 사람이다
    2. 회고 항목
        1. 배운점
        2. 아쉬운 점
        3. 개선사항
        4. 다시한다면?
6. 마무리 - 마지막 설득
- 입사 후 활용 가능성
- 핵심 역량 재 강조
- 이름 각인

인트로와 프로필 연결 전략

- 예를 들어, 인트로에서 문지 해결형 개발자 라고했으면 프로필에서는 그 근거가 보여야한다.

프로젝트 설명에 관한 올바른 예시

- 로그인 기능구현, JWT구현, DB설계, REST API 개발 → 기존 세션 방식에서 보안 취약점 발견하여 JWT 방식으로 전환 결정했고 토큰 만료 정책 설계하여 인증 오류율 35%감소
    - 무엇을 했는지 보다 왜 그렇게 했는지가 중요
- 팀원 들과 협업 했습니다. → 4인 팀 중 백엔드 전담, API 설계 및 DB 구조 책임 , 코드 리뷰 12회 진행, 트래픽 병목 해결 담당
    - 팀 프로젝트에서 가장 위험한 단어는 참여했습니다.
- 면접 질문으로 자연스럽기 이루어지겠금

프로젝트는 결과물이 아닌, 선택의 기록

- 문제 정의
- 전략
- 선택과 포기  이유
- 실패
- 회고

기업은 유연한 사고 방식, 문제 해결 능력, 함께 일할 수 있는 사람을 뽑는다.

컨셉과디자인

- 주요 정보는 중앙에 배치
- 요소 간 우선 순위를 명확히 구분
- 불필요한 정보는 최소화, 집중도를 높임
- 시각적 흐름을 고려하여 자연스러운 시선 이동 유도

톤 앤 매너의 일관성

- 폰트 동일
- 문장 톤 동일
- 컬러 동일
- 표현방식 동일
